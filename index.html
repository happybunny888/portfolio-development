Ticker's Tickets: Drive In Movie Theater Website

A drive in movie theater is a perfect ending to the day!

Development Project Description:
For this project, we were required to use React to develop a single page interactive interface, use interface components, and tie the components to an internal data state.



Interface Purpose:
The app is for users to purchase tickets to see movies at a drive in theater.
Users can see movies offered, movie information, and ticket price. Users can sort by ticket price or by ratings, and filter by movie type/genre.

(since this is mandated to be single page, did not consider ability to schedule).
(movie types available: action, romance, and anime).


Required Features

Description of functionality requirements and what I've done to fulfill those requirements are listed below.

At least two filtering categories:


  • action
  • anime
  • romance

At least 1 sorting feature:

  • price (lowest to highest)
  • rating (highest to lowest)

An aggregator section:

  • shopping cart: contains quantity and price of tickets for movie tickets added to cart.
  • aggregation: total price of all tickets in cart.
  • updates: updates based on changes made to cart.

At least 12 item cards containing:

  • image: movie poster
  • category: movie type/genre
  • sort fields: rating and price
  • property aggregated: price
  • aggregator changes: plus and minus button for adding and removing tickets.

Reset button for reverting to original state

Can handle combination of filters with sorting methods

Usability Requirements

Description of usability requirements and what I've done to fulfill those requirements are listed below.

Clear Status:

  • After selecting sort and filters: movies immediately presented in reflected order/filtering.
  • Increasing/decreasing/selecting tickets: immediately reflects items and pricing in cart.

Intuitive Use:

  • Sorting is intuitive: high to low for rating, and low to high for price.
  • Two clearly labelled buttons for changing quantities.

Visibility/Transparency:

  • Movie, quantity, price of tickets clearly displayed in the cart.
  • Filter options, sort options, reset filter button, reset cart button are visible in the app.
  • Movie items are clear, each movie item contains clearly labeled for ratings and types.
  • No short forms are used, all headings/labels are present.

Implementation Requirements

Description of implementation requirements and what I've done to fulfill those requirements are listed below.

Organized Components:


  • TicketItem Component: displays each movie/ticket item (title, description, price/ticket, Rotten Tomatoes rating, poster). Also contains buttons for adding and subtracting tickets to the cart for each movie.
  • Aggregator Component: accesses the state of the cart, displays the cart (items, quantity of tickets for each, price, and total price) and having a button to clear the shopping cart.
  • Filter Component: contains code for the function HandleChanged (selects the option of filtering from filter options, and changing the checked states of the filter), and contains code for display of the checkbox options for filter type.
  • Sort Component: contains code for the function HandleChanged (which selects the sorting type from the sort options, and changing the states of the sort), and contains code for display of sort options for sort type.

Data is Passed Down Through Components:


  • In App.js, map through all filtered/sorted movies which produce a TicketItem Component for each movie.
  • For each movie item mapped through: item, and the addToCart and removeFromCart functions are passed through the components.
  • There are two calls to state in App.js, one to retrieve the filter types and one to retrieve the sort.
  • The functions selectFilterType,selectSort, resetFilter,resetSort, matchesFilterType,matchesSortType are defined in App.js to help generate filtered/sorted mapping of movies on the tiles.
  • In the side bar area of the app in App.js, the components for Filter and Sort are present.

Changes to State:


  • when user increases/decreases quanity of ticket or clears cart.
  • when user changes filtering and/or sorting combinations, or resets.
